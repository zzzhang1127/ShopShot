# 正文：比赛完整要求

## 课题名称
电商场景 AIGC 带货视频生成系统

## 课题要求
在 TikTok Shop 国际电商快速发展的背景下，短视频已成为商家获取流量和提升成交的关键载体。本课题聚焦电商真实业务场景，邀请你基于（但不限于） React、Node.js、TypeScript、火山引擎 OpenAPI 与开源模型等，打造一个面向商家的 AIGC 带货视频生成系统。
带货视频生成系统希望可以实现商家端到端自动生成视频，支持素材导入与管理、 Prompt 调整、分镜干预、素材混剪等多项视频生成功能。课题不仅关注生成高质量的视频，也强调对电商交易转化效果，鼓励参赛者探索 AI Agent 在电商增长中的创新应用。

## 任务拆解

### 工程架构
- 前后端分离：采用前后端分离架构，如有大模型/Agent 编排诉求，后端可选用 LangChain、LangGraph 等开源框架，或者opencode/codex/claude code sdk等coding agent作为agent runtime。
- 代码仓库：项目整体代码存放于单一 Git 仓库，方便管理和协同。
- 编码规范：集成 ESLint、Prettier、Husky、StyleLint 等工具，确保代码风格统一和质量。
- 结构清晰：要求代码分层合理，模块化清晰，对"素材—剧本—创作"三大模块做好抽象，具备良好的可读性和可维护性。
- 能力对接：基于火山引擎 OpenAPI 调用文生图、文生视频、图生视频、TTS 等模型，并允许接入其他模型作为补充。

### 核心功能
核心流程：素材库建设 → 剧本生成 → 视频创作 → （非必要链路，辅助效果持续迭代：数据回流反哺）

#### 1. 素材模块
目标是给 AIGC 视频生成提供有效的素材，包括图像、视频片段。素材是一组可被调用的资产集合（而非单一图片或单一视频），用于提供商品真实外观、关键细节、尺度关系、正确使用方式等。
- 素材入库：支持商家手动上传商品主图、商品视频、参考素材等多类型素材。
- 多颗粒度结构化：对入库素材进行切片处理与多模态理解，自动提取商品维度（主体/类目）、视频维度（整体摘要/Embedding）、切片（slice）维度的细粒度特征，形成可被剧本/创作模块消费的结构化资产。
- 素材检索：提供基于关键词、标签、向量相似度的多颗粒度检索能力，支持下游剧本和创作的智能召回。

#### 2. 剧本模块
建设优质电商 AIGC 视频剧本的自动化生产能力，参考"找参考 → 提炼方法论 → 生产剧本"三步链路，沉淀优质视频库 / 模板 / 剧本三类核心实体。
- 优质视频库：按类目/关键词检索站外（FB/INS 等不限）爆款视频，每条视频配结构化拆解报告（Hook 手法、卖点、分镜、风格等），支持上传自有视频做拆解。
  - 只保存公开视频的结构化分析结果，不复刻、不混剪原视频；
  - 作业必须声明素材来源；
- 方法论提炼（灵感模板）：对一批同套路爆款视频做聚类（n:1），归纳为结构化创作模板，包含策略（视频创作的抽象方法，如"第一人称 BGM 氛围沉浸"）+ 因子（视频创作的具体手段，如"开场=轻柔音乐引入、退场=黑屏品牌名、画面重点=材料质感、旁白=优雅知性"）。
- 剧本生成：输入商品 ID/信息（标题、卖点、目标人群、场景），由"策略 + 因子 + 商品信息 + 约束规则"组合生成完整剧本，包含叙事框架、视觉风格、分镜脚本（含画面描述、镜头运动、BGM、字幕）、约束清单。
- 多种生产模式（不限于以下方式，可以自行创新）：
  - 爆款仿写：自动匹配最佳爆款（PID/类目逐级匹配）或人工指定参考视频，融合品信息生成 1:1 同款剧本。
  - 灵感模板：商家挑选模板 + 商品 ID → 融合出剧本。
  - 剧本自动化（进阶）：分层解耦"策略库 + 因子库 + 约束库"，按创作目标动态组合，每次产出多套风格各异的剧本。
- 剧本干预：支持用户对生成剧本进行 Prompt 微调、分镜增删、台词改写、因子局部替换（如视觉风格从"黑风"改为"夏日度假风"），并基于反馈快速重生成。

#### 3. 创作模块
根据素材和剧本进行视频创作，包括但不限于视频一键成片、智能剪辑等功能，视频时长不超过15s。
- 一键成片：用户仅需输入商品链接或上传商品主图，即可端到端产出一条完整带货视频；动线尽量与商家现有发文链路打通。
- 智能剪辑：基于素材切片三层标签体系（商品/视频/slice），结合剧本分镜，自动完成画面拼接、转场、字幕、配音（支持 polyglot/dubbing 多语种）、配乐合成。
- 分镜级干预：支持用户对单个分镜重生成、替换素材切片、调整时长、修改台词，干预后整体视频快速重渲染。
- 预览与导出：提供在线预览，支持导出多种分辨率与画幅（竖版 9:16、横版 16:9 等），适配不同分发渠道。

### 用户体验与前端挑战
- 首屏与交互：页面首屏快速加载，多模态内容（图片/视频/进度）渲染流畅，状态切换有清晰反馈。
- 长任务体验：视频生成属于长耗时任务，需设计合理的进度展示、断点续传、异常重试与失败兜底（生成失败/审核未通过的清晰反馈）。
- 分镜级编辑器：分镜列表 + 时间轴的复杂编辑交互需要良好的可视化与可操作性，支持拖拽、快捷键、实时预览，且能对单个分镜局部刷新而不重渲染整片。
- 数据看板：分析诊断模块需将"生成因子 × 转化效果"以直观方式呈现，前端需具备一定的数据可视化复杂度。
- 多端适配：考虑移动端商家在外的轻量编辑/查看体验，进行响应式布局设计。

## FAQ
- **技术栈必须严格遵守吗？** 核心技术栈（React、Node.js、TypeScript、火山引擎/外部模型 OpenAPI）建议保持一致；具体的 UI 组件库（Ant Design、Chakra、Shadcn 等）、图表库（ECharts、D3.js）、Agent 框架（LangChain/LangGraph、各种coding agent 等）以及开源模型可以自由选择，鼓励探索更合适的工具。
- **视频生成必须端到端跑通吗？** 基础要求是端到端可运行（哪怕单条链路、模型可替换为轻量版本）；视频效果不是评审重点，更看重工程链路的完整性、以及创新思路的探索。
- **数据从哪里来？** 分发与转化数据可使用 mock 数据或自建模拟服务，重点在于数据结构设计、归因逻辑与可视化呈现，而不是接入真实电商后台。
- **素材合规与版权如何处理？** 课题中可使用公开可商用素材或自有素材，重点是设计出"合规审核 + 真实性校验"的工作流，而非真实接入版权服务。
- **可以借用 AI 编程吗？** 完全可以，推荐使用 Trae 等 AI 编程工具，但需要对架构和代码质量做好把控。
- **可以使用第三方库吗？** 完全可以，开源社区的任何工具和方案都可以使用。
- **如何提交作业？** 给定可访问域名或本地启动访问，并附上 README、架构说明与生成效果 demo（建议附 1–2 条端到端生成的视频作品）。
- **可以组队吗？** 鼓励 2–3 人组队协作，最多不超过 3 人。建议分工明确，例如：素材+剧本链路、创作+智能剪辑、分发归因+数据看板分别由不同同学主攻。
